### Task 16 (S3b-1): Zonen-Zerlegung (pure lib)

**Files:** Create `src/diorama/ksw/interior/zones.ts`; Test `tests/interior/zones.test.ts`.

**Interfaces:**
- Produces: `decomposeToZones(footprint: number[][], opts?: { maxZones?: number; minSize?: number }): Zone[]` mit `Zone = { id: string; x: number; z: number; w: number; d: number }` (Achsparallele Rechtecke, Zentrum+Maße, Meter). Deterministisch: Grid-Rasterisierung des Polygons (Zellgröße 2 m, pointInRing auf Zellzentren) → wiederholt größtes achsparalleles Voll-Rechteck im Raster greedy entnehmen (Histogramm-Methode „largest rectangle in binary matrix", Standard-Algorithmus) bis `maxZones` (default 8) oder Restfläche < 15 %; Rechtecke < `minSize` (default 6 m Kantenlänge) verwerfen.
- Invarianten (Tests): Zonen überlappen nicht; jede Zone liegt vollständig im Polygon (alle 4 Ecken + Zentrum pointInRing); Abdeckung ≥ 60 % der Polygonfläche beim echten 113-Punkte-Footprint (aus `data/winterthur/buildings.json` laden — der Test läuft gegen ECHTE Daten); jede Zone ≥ 6×6 m; deterministisch (zweifacher Aufruf identisch).

**Steps:** TDD (Fixture: L-Polygon → erwartet 2 Zonen; dann Realdaten-Test) → implement → grün → Commit „interior(zones): rectilinear decomposition of the real KSW footprint".
